# 客語腔調與句子在地化架構

## 核心原則

- **遊戲判定使用 ID**：題庫中的 `target_outfit_ids` 只使用衣物與顏色 ID，不使用中文或任一腔調的文字。
- **畫面文字使用 locale**：衣物名稱、拼音、情境句與提示，均由在地化資料表依 `locale_code` 取得。
- **缺少翻譯時降級**：目標腔調 → 四縣腔 → 中文。

## 資料檔

| 檔案 | 用途 |
|---|---|
| `穿搭小達人 - 單字標籤.csv` | 衣物、顏色與資產資料；新增 `entity_id`。 |
| `穿搭小達人 - 出題架構.csv` | 關卡規則；新增 `target_outfit_ids`。 |
| `i18n/locales.csv` | 可用語言／腔調清單與啟用狀態。 |
| `i18n/vocabulary_localizations.csv` | 名稱與羅馬字；現有資料已建立中文與四縣腔列。 |
| `i18n/question_localizations.csv` | 情境句與錯誤提示；現有資料先建立中文列。 |

## 題目顯示規則

- 現階段：`sentenceLocale = zh-Hant`，句子維持中文；`vocabularyLocale = hak-sihien`，衣物可顯示四縣拼音。
- 未來切換客語句子：新增同一 `question_id` 的 `hak-<dialect>` 在地化列即可。
- 第 6–10 題要顯示拼音時，從 `vocabulary_localizations.csv` 讀取目標衣物或顏色的 `romanization`；不可由中文名稱推導。

## 新增腔調流程

1. 在 `locales.csv` 新增 `hak-<dialect>`。
2. 在 `vocabulary_localizations.csv` 補齊每個 `entity_id` 的名稱與羅馬字。
3. 若要使用該腔調句子，在 `question_localizations.csv` 補齊各題文字。
4. 將該腔調的 `enabled` 設為 `true`。
